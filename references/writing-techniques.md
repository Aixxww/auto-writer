# 写作技巧与钩子

从 baoyu-skills 中提取的写作风格技巧和前文钩子运用。

## 格式规范（去除 AI 痕迹）

### 禁止使用的格式符号
- 禁止使用 # 标题符号
- 禁止使用 --- 分割线
- 禁止使用 **加粗** 强调（仅限必要时使用）
- 禁止使用列表符号 - 开头的列表格式
- 禁止使用表格格式 |---|

### 推荐的天然格式
- 空行分隔段落
- 用数字或自然段落式的小标题（独立一行）
- 直接用句子叙述，不套用模板结构
- 标题仅用纯文字，不加符号
- 表格数据改用文字描述或绕口令式儿化表达

### 去除说教痕迹
- 禁止"我的建议""以下几点""首先是""其次"
- 禁止结尾总结性的"综上所述""总的来说"
- 禁止"让我们来看看""值得注意的是"
- 用故事、数据、案例代替说教
- 让读者自己得出结论，不要硬塞观点

## 开篇钩子技巧

### 1. 痛点钩子
直接指出读者的痛点或困惑：

```
"当你花了两小时写出的文章只有3个阅读时，你会怀疑自己。"
"有没有想过，为什么有些人的文章一发布就炸，而你的石沉大海？"
```

### 2. 数据钩子
用震撼的数据开篇：

```
"一条推文，24小时内带来500万次曝光。"
"DeepSeek-R2发布仅3小时，GitHub Star突破10万。"
```

### 3. 场景钩子
用具体场景吸引读者：

```
"凌晨三点，产品经理的微信突然亮了..."
"你刚刷完最后一条短视频，已经凌晨1点。"
```

### 4. 对比钩子
用强烈对比制造悬念：

```
"5年前，ChatGPT还需要排队申请；今天，每个人手机里都有AI。"
"有人花20万买流量，有人靠写作为零成本带来10万粉丝。"
```

### 5. 疑问钩子
用问题引发好奇：

```
"同样写大模型，为什么有人爆款不断，有人没人看？"
"DeepSeek-R2到底能不能打过GPT？答案可能和你想的不一样。"
```

### 6. 故事钩子
用小故事开场：

```
"去年11月，一个默默无闻的中国团队发布了DeepSeek-V2..."
"我认识一个程序员，去年还在纠结要不要跳槽..."
```

### 7. 金句钩子
用有冲击力的句子开篇：

```
"风口不是等来的，是看见的。"
"懂技术的人很多，能把技术讲清楚的人很少。"
```

## 内容结构技巧

### 递进式结构
一环扣一环，逐层深入：

```
Hook（钩子）→ Context（背景）→ Core（核心观点）→ Proof（论证）→ Action（行动建议）
```

### 对话式写法
模拟和读者对话，增加亲切感：

```
"你可能会问：这和我有什么关系？"
"这让我想起小时候..."
"别急，我们先看看..."
```

### 金币结构
每段结尾都有"金币"（高价值信息）：

```
段落1 → 金句/数据/洞察
段落2 → 另一个金币
段落3 → 再来一个
```

## 去AI味检查清单

### 格式层面

❌ 禁止的格式
- # 一级/二级标题符号 (改用纯文字独立成行)
- --- 水平分割线 (改用空行)
- **加粗强调** (除非必要，用句式传达重点)
- - 列表符号 (改为段落或数字叙述)
- | 表格 (改为文字描述)

✅ 推荐的格式
- 空行分隔自然段
- 小标题独立成行不加符号
- 用序数词或不加符号的段落
- 数据用叙述方式呈现

### 说教爹味清除

❌ 禁止说教痕迹
- "我的建议"、"以下几点"
- "首先"、"其次"、"最后"
- "综上所述"、"总的来说"
- "让我们来看看"、"值得注意的是"
- "作为一个XX，我认为"、"我想说"
- 结尾硬塞"总结"
- 硬升华到宏大叙事

✅ 推荐表达
- 用故事代替说教
- 让读者自己得出结论
- 不总结，用金句收尾
- 观点藏在叙事里
- 情绪自然流淌，不强行升华

### 句式层面

❌ 避免

```
- 首先、其次、然后、最后
- 综上所述、总而言之、值得一提
- 让我们来看看...、不难发现...
- 需要指出的是..., 值得注意的是...
- 过度使用排比句和工整对仗
- 全知视角，没有个人观点
- 每段都是总结式结尾
```

### ✅ 推荐

```
- 直接切入，不需要"首先"
- 口语化表达："其实..."、"说实话..."、"有意思的是..."
- 个人观点："在我看来..."、"我觉得..."、"如果是我..."
- 情绪表达："说实话我有点..."、"这让我想到..."
- 不完美句式："这个东西怎么说呢..."
- 短句增强节奏："就这么简单。"
- 问句引发思考："你猜怎么着？"
```

## 内容节奏技巧

### 情绪起伏
每篇文章至少有2-3个情绪高潮：

```
平静叙述 → 惊人数据（情绪高点）→ 继续叙述 → 金句共鸣（情绪高
点）→ 平静收尾
```

### 信息密度控制

| 部分 | 节奏 | 信息密度 |
|------|------|----------|
| 开头100字 | 快 | 高（钩子要快准狠）|
| 中间主体 | 中 | 中（每段1个核心点）|
| 结尾200字 | 慢 | 低（留白让读者思考）|

### 视觉休息点
每500-800字插入：
- 小标题（视觉分割）
- 数据/案例（打破纯文字）
- 金句/总结（强化印象）
- 问句（引发思考休息）

## 各领域写作风格

### 科技/AI领域

```
风格：专业但不说教
特点：
- 用类比解释复杂概念
- 数据支撑观点
- 用"我认为"增加可信度
- 避免"首先其次"的技术文档风
- 多用真实案例

示例：
"GPT-4的参数量没有公开，但根据测算可能在万亿级别——什么概念？
相当于把维基百科背100遍。" + 个人观点
```

### 财经领域

```
风格：数据驱动+个人判断
特点：
- 开头直接上数据
- 用历史对比增加厚度
- 明确区分"事实"和"观点"
- 给出具体可执行建议

示例：
"2024年A股涨幅前十的股票里，AI概念占了7席。
但这不代表现在追进去就是好主意——
在我看来，咱们散户更应该..."
```

### 情感领域

```
风格：真实+治愈
特点：
- 用具体场景代替抽象概念
- 不说教，只分享
- 留白让读者自己感悟
- 金句要真诚，不要鸡汤

示例：
"那天晚上，她问我：你觉得我们还有可能吗？
我没说话，但心里已经有了答案。
有时候，不回答就是最好的回答。"
```

### 知识科普领域

```
风格：趣味+系统
特点：
- 用故事或场景开篇
- 把复杂概念拆解成类比
- 每个知识点配一个案例
- 结尾升华或延伸

示例：
"你有没有想过，为什么熬夜之后明明很困却睡不着？
这和我们大脑里的'困素'有关——
简单来说，就像你家的电灯开关..."
```

## 标签使用技巧

### X平台标签（3-5个）

```
规则：
- 1个领域标签（#AI #财经）
- 2-3个关键词标签
- 最多5个，放文末

示例：
DeepSeek-R2深夜发布，性能直接对标GPT-4Turbo...
#AI #DeepSeek #大模型 #开源
```

### 微信公众号标签（3-5个）

```
规则：
- 放文章底部或顶部
- 1个领域 + 2-3个关键词
- 可加1个品牌标签

示例：
标签：#科技 #人工智能 #DeepSeek #大模型
```

## 配图文案技巧

### 封面图文案

```
X平台（5:2）：
- 大标题：核心关键词（字号大）
- 副标题：数据或亮点（字号小）
- 尽量简洁，突出一个要点

微信公众号（2.35:1）：
- 大标题：吸引眼球的短语
- 可加品牌或作者
- 底部留白给公众号名称
```

### 内图配文

```
数据图：直接标注关键数字
流程图：每步一个动词
概念图：用类比词（"相当于..."）
对比图：左右对比+差异标注
```
